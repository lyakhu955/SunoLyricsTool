📖 Guida Completa Suno

🎯 Come funziona Suno?

Suno ha due campi principali:

  • Style of Music → Definisce genere, strumenti, mood, era
  • Lyrics → Il testo della canzone con meta tags per controllare la struttura

📝 Regole d'Oro per i Lyrics

  • ✅ Mantieni le sezioni a 4 righe (Suno funziona meglio così)
  • ✅ Metti i meta tags all'inizio di ogni sezione
  • ✅ Usa parole semplici e chiare per una pronuncia migliore
  • ✅ Evita parole troppo lunghe o complesse
  • ✅ Usa [Instrumental] per le sezioni senza voce
  • ✅ Usa [End] o [Coda] per terminare la canzone
  • ❌ NON superare 6+ meta tags per sezione
  • ❌ NON mischiare generi incompatibili nei tag

🎛️ Formula Style Prompt

Segui questa struttura per lo Style Prompt:

decade, genere, sub-genere, paese, tipo voce, descrittori musicali

Esempio:

2010s, indie pop, rock, USA, female vocals, catchy hooks, melodic, polished production, rhythmic

🔑 Trucchi Pro dalla Community

  • 📌 Aggiungi [Produced by XXX] nei lyrics per influenzare lo stile di produzione
  • 📌 Usa tag di era come "80s", "90s" per ancorare il sound
  • 📌 Per realismo aggiungi: tape saturation, analog warmth, close mic presence, natural dynamics
  • 📌 Per vocal chiare usa: clear vocals, high-definition audio, wide stereo
  • 📌 Stile Formula: Core element → Genere/Era → Tone → Effects
  • 📌 Testa una modifica alla volta per capire cosa funziona
  • 📌 Le parentesi quadre [] possono essere "cantate" - usa formato metadata non frasi

⚠️ Errori Comuni

  • 🚫 Troppi tag → Suno si confonde (max 4-5 per sezione)
  • 🚫 Tag generici → "guitar" vs "60s jangly guitar rhythm"
  • 🚫 Mix generi incompatibili → [808 sub bass] + [pedal steel guitar]
  • 🚫 Lyrics box vuoto → Il prompt style potrebbe essere "cantato"
  • 🚫 Sezioni troppo lunghe → Più di 4-5 righe confonde il modello
